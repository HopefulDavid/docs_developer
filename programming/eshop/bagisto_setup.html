Bagisto instalace za pomocí dockeru

Základní informace

Na hostitelském systému musí být nainstalováno:

  • Docker Engine
  • Docker Compose v2

Ověření:

docker --version
docker compose version

Architektura řešení

  • PHP 8.2 + Apache běží v kontejneru
  • MySQL běží v samostatném kontejneru
  • Bagisto (Laravel) je uložen lokálně
  • Apache servíruje aplikaci z adresáře public
  • Databázová data jsou uložena v Docker volume

Struktura projektu

bagisto-docker/
├─ apache/
│  └─ vhost.conf
├─ workspace/
├─ Dockerfile
└─ docker-compose.yml

Složka workspace musí být při prvním spuštění prázdná.

Vytvoření souborů před instalací

Apache VirtualHost konfigurace

Laravel (a Bagisto) musí být servírován z adresáře public.

📄 apache/vhost.conf

<VirtualHost *:80>
    # Název serveru
    ServerName localhost
    
    # Kořenový adresář aplikace
    DocumentRoot /var/www/html/public

    # Nastavení přístupu ke složce public
    <Directory /var/www/html/public>
        # Povolit .htaccess soubory (např. pro Laravel
        AllowOverride All
        
        # Povolit přístup všem ke složce
        Require all granted
    </Directory>

    # Cesta k logům chyb
    ErrorLog ${APACHE_LOG_DIR}/error.log
    
    # Cesta k přístupovým logům
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>

Dockerfile (PHP 8.2 + požadované extensions)

Tento soubor vytvoříte v rootu projektu jako Dockerfile.

# Využíváme oficiální PHP image s Apache
FROM php:8.2-apache

# Instalace požadovaných PHP rozšíření a nástrojů
RUN apt-get update && apt-get install -y \
    git \
    zip \
    unzip \
    libzip-dev \
    libonig-dev \
    libpng-dev \
    libjpeg-dev \
    libfreetype6-dev \
    libxml2-dev \
    libicu-dev \
    && docker-php-ext-configure intl \
    && docker-php-ext-install \
        pdo_mysql \
        mbstring \
        zip \
        exif \
        pcntl \
        bcmath \
        gd \
        intl \
        calendar

# Povolení mod_rewrite pro Apache
RUN a2enmod rewrite

# Kopírování vlastní Apache konfigurace
COPY apache/vhost.conf /etc/apache2/sites-available/000-default.conf

# Instalace Composeru
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

# Nastavení pracovního adresáře
WORKDIR /var/www/html

# Nastavení časové zóny na UTC
RUN echo "date.timezone=UTC" > /usr/local/etc/php/conf.d/timezone.ini

Docker environment konfigurace

Tento soubor vytvoříte v rootu projektu jako .env.

# =====================
# Aplikace
# =====================
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

# =====================
# Databáze
# =====================
DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=bagisto
DB_USERNAME=bagisto
DB_PASSWORD=secret

# =====================
# MySQL nastavení
# =====================
MYSQL_ROOT_PASSWORD=root
MYSQL_DATABASE=bagisto
MYSQL_USER=bagisto
MYSQL_PASSWORD=secret

# =====================
# Porty
# =====================
APP_PORT=8000
PHPMYADMIN_PORT=8081
MYSQL_PORT=3306

Docker Compose konfigurace

Tento soubor vytvoříte v rootu projektu jako docker-compose.yml.

services:

  app:
    build:
      context: .
      dockerfile: Dockerfile
    image: bagisto-app
    container_name: bagisto_app
    ports:
      - "${APP_PORT}:80"
    volumes:
      - ./workspace:/var/www/html
    environment:
      APP_ENV: ${APP_ENV}
      APP_DEBUG: ${APP_DEBUG}
      APP_URL: ${APP_URL}

      DB_CONNECTION: ${DB_CONNECTION}
      DB_HOST: ${DB_HOST}
      DB_PORT: ${DB_PORT}
      DB_DATABASE: ${DB_DATABASE}
      DB_USERNAME: ${DB_USERNAME}
      DB_PASSWORD: ${DB_PASSWORD}
    depends_on:
      - mysql

  mysql:
    image: mysql:8.0
    container_name: bagisto_mysql
    restart: unless-stopped
    ports:
      - "${MYSQL_PORT}:3306"
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    volumes:
      - ./Volumes/mysql-data:/var/lib/mysql

  phpmyadmin:
    image: phpmyadmin/phpmyadmin
    container_name: bagisto_phpmyadmin
    restart: unless-stopped
    ports:
      - "${PHPMYADMIN_PORT}:80"
    environment:
      PMA_HOST: ${DB_HOST}
      PMA_PORT: ${DB_PORT}
      PMA_USER: ${DB_USERNAME}
      PMA_PASSWORD: ${DB_PASSWORD}
    depends_on:
      - mysql

Spuštění Docker prostředí

Uvnitř root složky projektu spusťte:

docker compose up -d --build

Po dokončení běží:

  • PHP/Apache kontejner
  • MySQL kontejner

Vytvoření Bagisto projektu

Projekt se vytváří výhradně uvnitř PHP kontejneru.

docker exec -it bagisto_app bash
composer create-project bagisto/bagisto .

Po dokončení jsou soubory okamžitě dostupné ve složce workspace.

Instalace Bagista

php artisan bagisto:install

Instalátor vás provede několika kroky.

Správné databázové hodnoty

Database connection: mysql
Database host: mysql
Database port: 3306
Database name: bagisto
Database username: bagisto
Database password: secret

Instalátor:

  • vytvoří .env soubor pro Laravel s potřebnými hodnotami
  • vytvoří databázové tabulky
  • naplní výchozí data
  • vytvoří admin účet

Nastavení oprávnění (NUTNÉ)

Laravel vyžaduje zapisovat do storage a bootstrap/cache.

chown -R www-data:www-data storage bootstrap/cache
chmod -R 775 storage bootstrap/cache
php artisan optimize:clear

Tento krok je zásadní zejména na Windows a macOS.

Nyní restartujte kontejner:

docker compose restart app

Přístup k aplikaci

  • Frontend: http://localhost:8000

  • Administrace: http://localhost:8000/admin

Instalace XDebugu (volitelné)

Pokud chcete ladit aplikaci, můžete doinstalovat XDebug.

Vytvoření konfiguračního souboru

Konfiguraci vytvořte v /apache/conf.d/xdebug.ini

xdebug.mode=debug
xdebug.start_with_request=yes

xdebug.client_host=host.docker.internal
xdebug.client_port=9003

xdebug.idekey=PHPSTORM

Připojení konfigurace do kontejneru

V docker-compose.yaml u služby app přidej volume:

services:
  app:
    volumes:
      - ./apache/conf.d/xdebug.ini:/usr/local/etc/php/conf.d/xdebug.ini

Instalace XDebugu přes Dockerfile

V Dockerfile přidáme instalaci XDebugu:

# Využíváme oficiální PHP image s Apache
FROM php:8.2-apache

# Instalace požadovaných PHP rozšíření a nástrojů
RUN apt-get update && apt-get install -y \
    git \
    zip \
    unzip \
    libzip-dev \
    libonig-dev \
    libpng-dev \
    libjpeg-dev \
    libfreetype6-dev \
    libxml2-dev \
    libicu-dev \
    && docker-php-ext-configure intl \
    && docker-php-ext-install \
        pdo_mysql \
        mbstring \
        zip \
        exif \
        pcntl \
        bcmath \
        gd \
        intl \
        calendar

# ZDE PŘIDAT: Instalace Xdebug
RUN pecl install xdebug \
    && docker-php-ext-enable xdebug

# Povolení mod_rewrite pro Apache
RUN a2enmod rewrite

# Kopírování vlastní Apache konfigurace
COPY apache/vhost.conf /etc/apache2/sites-available/000-default.conf

# Instalace Composeru
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

# Nastavení pracovního adresáře
WORKDIR /var/www/html

# Nastavení časové zóny na UTC
RUN echo "date.timezone=UTC" > /usr/local/etc/php/conf.d/timezone.ini

Nyní provedeme rebuild kontejneru:

docker-compose down
docker-compose build --no-cache
docker-compose up -d

Protože jsme změnili Dockerfile (přidali instalaci Xdebugu), Docker automaticky znovu sestaví potřebné vrstvy a Xdebug se nainstaluje.

Parametr --no-cache není v tomto případě nutný, ale lze ho použít jako pojistku, pokud si chceme být 100 % jisti, že se nepoužije žádná stará cache.

Best practices

  • Composer se spouští pouze v kontejneru
  • Zdrojový kód se edituje lokálně
  • Databázový host je vždy název služby (mysql)
  • Laravel se servíruje pouze z public
  • Docker řeší infrastrukturu, Laravel aplikační logiku
